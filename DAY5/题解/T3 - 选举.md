# 让我们赢得选举

## 题目大意

有 $N$ 个州。第 $i$ 个州累计演讲时间达到 $A_i$ 时，可以获得这个州的一张选票；如果累计演讲时间达到 $B_i$，还可以获得一名协作者。若 $B_i=-1$，表示这个州无法获得协作者。

一开始只有 Rie 一个人可以演讲，每获得一名协作者，就多一个人可以同时在其他州演讲。多个人可以同时演讲，所以如果当前有 $p$ 个人，总共完成 $x$ 小时的演讲量，只需要花费 $\frac{x}{p}$ 小时。

现在要求最短多少时间可以得到至少 $K$ 张选票。

## 思路分析

首先看几个比较直接的情况。

如果所有 $B_i=-1$，也就是完全没有协作者，那么只能靠 Rie 一个人逐个州演讲。此时直接选择 $A_i$ 最小的 $K$ 个州即可，可以拿到子任务 $1$ 的 $5$ 分。

如果 $N$ 很小，可以枚举哪些州拿来获得协作者、哪些州只拿选票，再枚举协作者的获得顺序。这样能处理 $N\le 7$ 或 $N\le 20$ 一类的小数据，可以拿到对应的部分分，但显然无法通过 $N\le 500$ 的完整数据。

下面考虑正解。

我们先固定最终要获得 $x$ 个协作者。由于获得一个协作者的同时也会获得这个州的选票，所以这 $x$ 个州本身也贡献 $x$ 张选票，剩下还需要再获得 $K-x$ 张普通选票。

如果当前已经有 $j$ 个人，那么获得下一个协作者需要花费：

$$
\frac{B_i}{j}
$$

假设已经确定了要从若干个州获得协作者，那么这些州一定应该按照 $B_i$ 从小到大的顺序处理。因为对于两个协作者州 $u,v$，若 $B_u\le B_v$，先处理 $u$ 再处理 $v$ 的花费为：

$$
\frac{B_u}{j}+\frac{B_v}{j+1}
$$

反过来的花费为：

$$
\frac{B_v}{j}+\frac{B_u}{j+1}
$$

前者不会更差，所以协作者州按 $B_i$ 升序处理最优。

于是我们把所有州按照 $B_i$ 从小到大排序，不能获得协作者的州可以看成 $B_i=\infty$，放在后面。

接下来还有一个关键性质：对于固定的 $x$，如果某个州被选作协作者，并且在排序后它前面有一个州没有被选择，那么我们可以把这个协作者替换成前面那个州。因为前面那个州的 $B$ 不会更大，获得协作者的代价不会变差，选票数量也不变。

所以在最优方案中，设最后一个被选作协作者的州在排序后的位置为 $i$，那么前 $i$ 个州一定都会被选择：其中 $x$ 个作为协作者，其余作为普通选票。剩下还差的选票，就从后缀里选择 $A_i$ 最小的若干个州即可。

因此可以这样做：

1. 按 $B_i$ 升序排序；
2. 预处理 `suf[i][j]`，表示从排序后第 $i$ 个州到第 $N$ 个州中，选择 $j$ 个普通选票所需的最小 $A$ 之和；
3. 枚举最终协作者数量 $x$；
4. 用 DP 计算前缀中全都被选择，并且其中有若干个协作者时的最小花费。

设 $dp[i][j]$ 表示考虑排序后的前 $i$ 个州，这 $i$ 个州全部被选择，并且其中有 $j$ 个州作为协作者时的最小花费。

对于第 $i$ 个州，有两种选择：

- 把它作为普通选票，花费为 $\frac{A_i}{x+1}$，因为普通选票可以等所有 $x$ 个协作者都获得之后再去完成；
- 把它作为第 $j$ 个协作者，花费为 $\frac{B_i}{j}$。

状态转移为：

$$
dp[i][j]=\min(dp[i][j],dp[i-1][j]+\frac{A_i}{x+1})
$$

以及：

$$
dp[i][j]=\min(dp[i][j],dp[i-1][j-1]+\frac{B_i}{j})
$$

处理完前缀后，假设前缀长度为 $i$，其中已经有 $x$ 个协作者，那么前缀本身贡献了 $i$ 张选票，还需要从后缀中再选 $K-i$ 张普通选票，花费为：

$$
\frac{suf[i+1][K-i]}{x+1}
$$

枚举所有 $x$ 和 $i$，取最小值即可。

下面是正解：

```c++
#include <bits/stdc++.h>
using namespace std;

const double INF = 1e100;
const long long LINF = (1LL << 60);

struct State {
    int a, b;
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, k;
    cin >> n >> k;

    vector<State> p(n + 1);
    int can = 0;
    for (int i = 1; i <= n; i++) {
        cin >> p[i].a >> p[i].b;
        if (p[i].b == -1) {
            p[i].b = 1000000000;
        } else {
            can++;
        }
    }

    sort(p.begin() + 1, p.end(), [](const State &x, const State &y) {
        return x.b < y.b;
    });

    // suf[i][j]：从 i..n 中选 j 个州作为普通选票，最小 A 之和
    vector<vector<long long>> suf(n + 2, vector<long long>(k + 1, LINF));
    suf[n + 1][0] = 0;
    for (int i = n; i >= 1; i--) {
        for (int j = 0; j <= k; j++) {
            suf[i][j] = min(suf[i][j], suf[i + 1][j]);
            if (j > 0 && suf[i + 1][j - 1] != LINF) {
                suf[i][j] = min(suf[i][j], suf[i + 1][j - 1] + p[i].a);
            }
        }
    }

    double ans = INF;

    for (int x = 0; x <= min(can, k); x++) {
        vector<vector<double>> dp(n + 1, vector<double>(x + 1, INF));
        dp[0][0] = 0;

        for (int i = 1; i <= n; i++) {
            for (int j = 0; j <= min(i, x); j++) {
                // 当前州作为普通选票
                dp[i][j] = min(dp[i][j], dp[i - 1][j] + 1.0 * p[i].a / (x + 1));

                // 当前州作为第 j 个协作者
                if (j > 0 && p[i].b != 1000000000) {
                    dp[i][j] = min(dp[i][j], dp[i - 1][j - 1] + 1.0 * p[i].b / j);
                }
            }
        }

        for (int i = x; i <= k; i++) {
            int need = k - i;
            if (need < 0 || need > n - i) continue;
            if (dp[i][x] == INF || suf[i + 1][need] == LINF) continue;

            ans = min(ans, dp[i][x] + 1.0 * suf[i + 1][need] / (x + 1));
        }
    }

    cout << fixed << setprecision(15) << ans << '\n';

    return 0;
}
```

预处理后缀最小选票和需要 $O(NK)$。枚举协作者数量 $x$，每次 DP 复杂度为 $O(Nx)$，总复杂度为：

$$
O(NK+N\sum_{x=0}^{K}x)=O(NK^2)
$$

由于 $N\le 500$，这个复杂度可以通过。空间复杂度为 $O(NK)$。
