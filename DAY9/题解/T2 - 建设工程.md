# 建设工程

## 题目大意

给定一张 $N$ 个点、$M$ 条边的无向带权图。现在可以选择一对点 $u<v$，新建一条长度为 $L$ 的双向边。

问有多少种选择 $(u,v)$，使得加上这条边之后，从 $S$ 到 $T$ 的最短路长度不超过 $K$。

## 思路分析

首先考虑直接做法。对于每一对 $(u,v)$ 都尝试加边，然后重新求一次最短路，这样复杂度肯定无法接受。即使先求出所有点对最短路，再枚举所有 $(u,v)$，也至少需要 $O(N^2)$ 的枚举，只能通过 $N\le 3000$ 左右的部分数据。

我们考虑一条新边真正会怎样出现在 $S$ 到 $T$ 的路径中。

如果新建边连接 $u,v$，那么经过这条新边的路径只有两种方向：

$$
S\to u\to v\to T
$$

或者：

$$
S\to v\to u\to T
$$

所以我们只需要知道：

- $ds[x]$：从 $S$ 到 $x$ 的最短路；
- $dt[x]$：从 $T$ 到 $x$ 的最短路。

这两个数组可以分别从 $S$ 和 $T$ 跑一次 Dijkstra 得到。

如果原本就有：

$$
ds[T]\le K
$$

那么不管新边建在哪里，国王都会高兴，答案直接是：

$$
\frac{N(N-1)}2
$$

否则必须依赖新建边。对于一个方向 $u\to v$，需要满足：

$$
ds[u]+L+dt[v]\le K
$$

也就是：

$$
dt[v]\le K-L-ds[u]
$$

于是可以把所有有限的 $dt[v]$ 排序。枚举 $u$，二分统计有多少个 $v$ 满足上面的不等式。

这里还有一个细节：题目要求的是无序点对 $(u,v)$，但我们枚举的是方向 $u\to v$。为什么不会算重？

因为我们已经处理掉了 $ds[T]\le K$ 的情况。现在有 $ds[T]>K$。如果对于同一对不同点 $u,v$，两个方向都成立：

$$
ds[u]+L+dt[v]\le K
$$

并且：

$$
ds[v]+L+dt[u]\le K
$$

两式相加可得：

$$
(ds[u]+dt[u])+(ds[v]+dt[v])+2L\le 2K
$$

而对任意点 $x$，都有：

$$
ds[x]+dt[x]\ge ds[T]
$$

所以会推出：

$$
ds[T]\le K-L<K
$$

这和当前假设 $ds[T]>K$ 矛盾。因此同一个无序点对最多只有一个方向会被统计到。

同时，$u=v$ 也不会被错误统计。若 $ds[u]+L+dt[u]\le K$，同样会推出 $ds[T]\le K-L<K$，矛盾。

所以枚举有向方向得到的数量，正好就是合法无序点对数量。

下面是正解：

```c++
#include <bits/stdc++.h>
using namespace std;

using ll = long long;

const ll INF = (ll)4e18;

struct Edge {
    int to;
    ll w;
};

void dijkstra(int s, const vector<vector<Edge>> &g, vector<ll> &dist) {
    int n = (int)g.size() - 1;
    dist.assign(n + 1, INF);

    priority_queue<pair<ll, int>, vector<pair<ll, int>>, greater<pair<ll, int>>> pq;
    dist[s] = 0;
    pq.push({0, s});

    while (!pq.empty()) {
        auto [d, u] = pq.top();
        pq.pop();

        if (d != dist[u]) continue;

        for (auto e : g[u]) {
            if (dist[e.to] > d + e.w) {
                dist[e.to] = d + e.w;
                pq.push({dist[e.to], e.to});
            }
        }
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, m;
    cin >> n >> m;

    int S, T;
    ll L, K;
    cin >> S >> T >> L >> K;

    vector<vector<Edge>> g(n + 1);
    for (int i = 0; i < m; i++) {
        int u, v;
        ll w;
        cin >> u >> v >> w;
        g[u].push_back({v, w});
        g[v].push_back({u, w});
    }

    vector<ll> ds, dt;
    dijkstra(S, g, ds);
    dijkstra(T, g, dt);

    if (ds[T] <= K) {
        cout << 1LL * n * (n - 1) / 2 << '\n';
        return 0;
    }

    vector<ll> vals;
    for (int i = 1; i <= n; i++) {
        if (dt[i] != INF) {
            vals.push_back(dt[i]);
        }
    }
    sort(vals.begin(), vals.end());

    ll ans = 0;

    for (int u = 1; u <= n; u++) {
        if (ds[u] == INF) continue;

        ll limit = K - L - ds[u];
        if (limit < 0) continue;

        ans += upper_bound(vals.begin(), vals.end(), limit) - vals.begin();
    }

    cout << ans << '\n';

    return 0;
}
```

两次 Dijkstra 的复杂度为 $O((N+M)\log N)$，排序和枚举二分复杂度为 $O(N\log N)$，所以总复杂度为 $O((N+M)\log N)$。空间复杂度为 $O(N+M)$。
