# T118342.勇者比太郎

## 题目描述

输入文件：`hero.in`  
输出文件：`hero.out`

比太郎要打怪。

令比太郎初始时的力量为 `x`。有 `N` 个怪物，编号 `1∼N`。欲打败第 `i`（`1≤i≤N`）个怪物，需要力量 `≥Ai`。打败第 `i`（`1≤i≤N`）个怪物，会使比太郎的力量增加 `Bi`。

比太郎会用如下的策略打怪：

1. 选择整数 `j`（`1≤j≤N`），然后按 `j, j+1, ..., N` 的顺序打怪。
2. 如果 `j≥2`，回头按顺序打怪物 `1,2,...,j−1`。

在按照策略打完所有的怪物的前提下，求出比太郎初始力量 `x` 的最小值。

## 输入格式

第一行包含一个整数 `N`。

第二行包含用空格分隔的 `N` 个整数 `A1, A2, ..., AN`。

第三行包含用空格分隔的 `N` 个整数 `B1, B2, ..., BN`。

## 输出格式

输出一行一个整数，即比太郎初始力量 `x` 的最小值。

## 输入输出样例

### 输入 #1

```text
5
1 3 2 8 6
4 3 1 1 2
```

### 输出 #1

```text
1
```

```

### 输出 #2

3
```

### 输入 #3

### 输出 #3

9
```

### 输入 #4

```

### 输出 #4

```text
0
```

## 说明 / 提示

### 样例解释

样例 1 解释：

令 `x=1`，然后按照 `1→5` 的顺序打怪。

该样例满足子任务 `1,2,3,5` 的限制。

样例 2 解释：

令 `x=3`，然后先打 `3→5` 的怪，再打 `1→2` 的怪。

该样例满足子任务 `1,2,3,5` 的限制。

样例 3 解释：

该样例满足所有子任务的限制。

样例 4 解释：

该样例满足子任务 `1,2,3,5` 的限制。

## 数据范围

`2≤N≤5×10^5`。

`0≤Ai≤10^9`（`1≤i≤N`）。

`0≤Bi≤10^9`（`1≤i≤N`）。

输入的值全部是整数。

## 子任务

1. `1∼5`：`10` 分，`N≤2000`，保证答案不大于 `10`
2. `6∼15`：`20` 分，`N≤2000`
3. `16∼25`：`20` 分，保证答案不大于 `10`
4. `26∼35`：`20` 分，`Bi=1`（`1≤i≤N`）
5. `36∼50`：`30` 分，无额外限制
