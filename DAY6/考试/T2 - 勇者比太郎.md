# 勇者比太郎

输入文件：hero.in

输出文件：hero.out

## 题目描述

比太郎要打怪。

令比太郎初始时的力量为 $x$。有 $n$ 个怪物，编号为 $1 \sim n$。若要打败第 $i$ 个怪物，需要当前力量满足 $x \geq a_i$。打败第 $i$ 个怪物后，比太郎的力量会增加 $b_i$。

比太郎会使用如下策略打怪：

1. 选择一个整数 $j$，满足 $1 \leq j \leq n$，然后按 $j,j+1,\dots,n$ 的顺序打怪；
2. 如果 $j \geq 2$，再回头按 $1,2,\dots,j-1$ 的顺序打怪。

在能够按照上述策略打败所有怪物的前提下，请求出比太郎初始力量 $x$ 的最小值。

## 输入格式

输入的第一行包含一个整数 $n$。

输入的第二行包含 $n$ 个整数 $a_1,a_2,\dots,a_n$。

输入的第三行包含 $n$ 个整数 $b_1,b_2,\dots,b_n$。

## 输出格式

输出一行一个整数，表示比太郎初始力量 $x$ 的最小值。

## 样例

### 样例 1 输入

```text

```

### 样例 1 输出

```text
1
```

```text
9
```

```text
0
```

## 提示

### 【样例 1 解释】

令 $x=1$，选择 $j=1$，然后按照 $1,2,\dots,5$ 的顺序打怪，可以打败所有怪物。

该样例满足子任务 $1,2,3,5$ 的限制。

### 【样例 2 解释】

令 $x=3$，选择 $j=3$，然后先打第 $3,4,5$ 个怪物，再打第 $1,2$ 个怪物，可以打败所有怪物。

该样例满足子任务 $1,2,3,5$ 的限制。

### 【样例 3 解释】

该样例满足所有子任务的限制。

### 【样例 4 解释】

该样例满足子任务 $1,2,3,5$ 的限制。

### 【数据范围】

本题采用测试点捆绑，同一行中的测试点作为一个测试点组计分。

对于所有测试数据，保证：

- $2 \leq n \leq 5 \times 10^5$；
- $0 \leq a_i \leq 10^9$；
- $0 \leq b_i \leq 10^9$；
- 输入的所有数均为整数。

| 子任务 | 分值 | $n$ | 特殊性质 |
| :--: | :--: | :--: | :--: |
| $1 \sim 5$ | $10$ | $2000$ | A |
| $6 \sim 15$ | $20$ | $2000$ | 无 |
| $16 \sim 25$ | $20$ | $5 \times 10^5$ | A |
| $26 \sim 35$ | $20$ | $5 \times 10^5$ | B |
| $36 \sim 50$ | $30$ | $5 \times 10^5$ | 无 |

特殊性质 A：保证答案不大于 $10$。

特殊性质 B：对于所有 $1 \leq i \leq n$，均有 $b_i=1$。
