# 修复二进制串

## 题目描述

给定一个长度为 $n$ 的二进制字符串 $s$，由 0 和 1 组成。你可以执行以下操作**恰好一次**：

1. 选择一个整数 $p$（$1 \le p \le n$）；
2. 反转子串 $s_1 s_2 \ldots s_p$。执行此步骤后，字符串 $s_1 s_2 \ldots s_n$ 将变为 $s_p s_{p-1} \ldots s_1 s_{p+1} s_{p+2} \ldots s_n$；
3. 然后，将字符串 $s$ 循环左移 $p$ 次。执行此步骤后，初始字符串 $s_1s_2 \ldots s_n$ 将变为 $s_{p+1}s_{p+2} \ldots s_n s_{p-1} \ldots s_1$。

例如，对字符串 `110001100110` 执行 $p=3$ 的操作：
- 第二步后变为 `011001100110`；
- 第三步后变为 `001100110011`。

称字符串 $s$ 为 $k$-proper，当且仅当满足以下两个条件：
- $s_1 = s_2 = \ldots = s_k$；
- 对所有 $i$（$1 \le i \le n - k$），有 $s_{i+k} \neq s_i$。

当 $k=3$ 时：
- `000`、`111000111` 和 `111000` 是 $k$-proper 的；
- `000000`、`001100` 和 `1110000` 不是 $k$-proper 的。

给定整数 $k$（保证是 $n$ 的约数），找出一个满足 $1 \le p \le n$ 的整数 $p$，使得执行操作后字符串 $s$ 变为 $k$-proper。如果不可能实现，请判定为无解。  
**注意**：即使初始字符串已经是 $k$-proper，仍需执行恰好一次操作。

## 输入格式

每个测试包含多个测试用例。第一行输入一个整数 $t$（$1 \le t \le 10^4$）—— 表示测试用例的数量。接下来是每个测试用例的描述。

每个测试用例的第一行包含两个整数 $n$ 和 $k$（$1 \le k \le n$，$2 \le n \le 10^5$）—— 分别表示字符串 $s$ 的长度和参数 $k$ 的值。保证 $k$ 是 $n$ 的**约数**。

每个测试用例的第二行包含一个长度为 $n$ 的二进制字符串 $s$，由字符 0 和 1 组成。

保证所有测试用例的 $n$ 之和不超过 $2 \cdot 10^5$。

## 输出格式

对于每个测试用例，输出一个整数：
- 若能通过操作使字符串变为 $k$-proper，输出满足条件的 $p$ 值（$1 \le p \le n$）；
- 若不可能实现，输出 $-1$。

若存在多个可行的 $p$ 值，输出其中任意一个即可。

## 说明/提示

#### 【样例 1 解释】

在第一个测试用例中，若选择 $p=3$ 执行操作：
- 第二步操作后字符串变为 **111**00001
- 第三步操作后字符串变为 00001**111**
最终字符串满足 $4$-proper 条件

在第三个测试用例中，若选择 $p=7$ 执行操作：
- 第二步操作后字符串变为 **1000111**00011
- 第三步操作后字符串变为 00011**1000111** 
最终字符串满足 $3$-proper 条件

## 题目大意

给定一个长度为 $n$ 的二进制字符串 $s$，称字符串 $s$ 为 $k$-proper，当且仅当满足以下两个条件：
- $s_1 = s_2 = \ldots = s_k$；
- 对所有 $i$（$1 \le i \le n - k$），有 $s_{i+k} \neq s_i$。

可以执行一次操作：

1. 选择一个整数 $p$（$1 \le p \le n$）；
2. 反转子串 $s_1 s_2 \ldots s_p$。执行此步骤后，字符串 $s_1 s_2 \ldots s_n$ 将变为 $s_p s_{p-1} \ldots s_1 s_{p+1} s_{p+2} \ldots s_n$；
3. 然后，将字符串 $s$ 循环左移 $p$ 次。执行此步骤后，初始字符串 $s_1s_2 \ldots s_n$ 将变为 $s_{p+1}s_{p+2} \ldots s_n s_p s_{p-1} \ldots s_1$。

给定 $k$，问能否经过一次操作把这个字符串变成 $k$-proper 的字符串。

## 思路分析

首先是 $n <= 1000$，这个时候我们可以对字符串的每一个位置暴力执行一次操作，看执行操作之后是否是满足条件的，这样的做法是 $O(n^2)$ 的，可以拿到 30 分。

```c++
void solve() {
    int n, k; cin >> n >> k;
    string s; cin >> s;
    for (int i = 1; i <= n ; ++i) {
        string tmp = s;
        reverse(tmp.begin(), tmp.begin() + i);
        for (int j = 1; j <= i ; ++j) {
            tmp += tmp[0];
            tmp.erase(0,1);
        }
        bool flag = false;
        for (int j = 0; j < k - 1; ++j) {
            if(tmp[j] != tmp[j+1]){
                flag = true;
                break;
            }
        }
        for (int j = 0; j < n - k ; ++j) {
            if(tmp[j+k] == tmp[j]){
                flag = true;
                break;
            }
        }
        if(!flag){
            cout << i << endl;
            return;
        }
    }
    cout << -1 << endl;
}
```

考虑 $k=1$ 的性质，这个时候如果字符串想要满足条件，一定是形如 $01010110101$，其中只能存在一对相邻的 $0$ 或 $1$，对这个前缀进行操作之后检查是否满足条件，把这个特殊性质写成特判，和暴力的代码加在一起，理论上可以拿到 $50$ 分。

```c++
void solve() {
    int n, k; cin >> n >> k;
    string s; cin >> s;
    if(k==1){
        int tmp = -1;
        for(int i=0;i<n-1;i++){
            if(s[i] == s[i+1]){
                tmp = i + 1;
            }
        }
        if(tmp==-1){
            cout << n << endl;
            return;
        }
        reverse(s.begin(), s.begin() + tmp);
        for (int j = 1; j <= tmp ; ++j) {
            s += s[0];
            s.erase(0,1);
        }
        bool flag = false;
        for (int j = 0; j < n - 1; ++j) {
            if(s[j] == s[j+1]){
                flag = true;
                break;
            }
        }
        if(!flag){
            cout << tmp << endl;
        }else{
            cout << -1 << endl;
        }
        return;
    }
}
```

我们考虑发现，假设我们现在从前向后遍历这个字符串，判断其前缀是否满足条件，当我们找到一个不满足条件的地方的时候，直接对这个前缀执行题目中所说的操作。如果操作后字符串满足条件，那么这个就是解，否则不存在解。

这个东西可以这样来理解，如果我们从前向后找到了一个不合法的状况并且忽略掉，我们往后又找到了一个操作的区间，那么我们操作之后，之前不合法的位置依然存在，所以我们只考虑第一个找到的不合法的位置即可。

```c++
void solve() {
    int n, k; cin >> n >> k;
    string s; cin >> s;
    int cnt = 0, cur = s[0] - '0';
    int ans = -1;
    for (int i = 0; i < n; i++) {
        if (cur == s[i] - '0') cnt++;
        else {
            if (cnt == k) {
                cur ^= 1;
                cnt = 1;
            } else {
                if (cnt < k) ans = i;
                else ans = i - k;
                break;
            }
        }
    }
    if (ans == -1 && cnt != k) {
        cout << -1 << endl; return;
    }
    if (ans == -1) {
        cout << n << endl; return;
    }
    reverse(s.begin(), s.begin() + ans);
    for (int i = 1; i <= ans; i++) {
        s += s[0];
        s.erase(0, 1);
    }
    for (int i = 0; i < k; i++) {
        if (s[i] != s[0]) {
            cout << -1 << endl;
            return;
        }
    }
    for (int i = 0; i < n - k; i++) {
        if (s[i] == s[i + k]) {
            cout << -1 << endl;
            return;
        }
    }
    cout << ans << endl;
}
```
