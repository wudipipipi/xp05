# 口算比赛

## 题目大意

给定一个长度为 $n$ 的数组，需要依次执行 $q$ 条指令：

- `1 i x`：把当前序列的第 $i$ 个数改成 $x$；
- `2 k`：把当前序列循环右移 $k$ 位。

每执行完一条指令，都要输出当前整个序列的元素和。

## 思路分析

首先我们考虑直接模拟。如果是第一类操作，直接修改第 $i$ 个位置即可；如果是第二类操作，就真的把整个数组循环右移。这样一次移动可能需要 $O(n)$，而 $n,q$ 都可以达到 $10^5$，最坏复杂度会到 $O(nq)$，显然不能通过。

如果所有操作都是第一类，那么我们只需要维护总和，每次修改一个位置即可，这样可以拿到子任务 $1$ 的 $22$ 分。如果 $n\le 1000$，直接模拟循环移动也可以接受，可以通过前三个子任务，一共拿到 $22+17+23=62$ 分。

注意到题目每次只要求输出总和，而循环移动并不会改变数组中有哪些数，所以第二类操作不会改变总和。

真正需要处理的是第一类操作：经过若干次循环右移之后，当前序列的第 $i$ 个位置，实际上对应原数组中的哪个位置？

我们用一个变量 $shift$ 表示当前序列相对于原数组一共向右移动了多少位。为了避免数字过大，可以始终对 $n$ 取模。

如果当前已经整体右移了 $shift$ 位，那么当前第 $i$ 个位置对应原数组中的位置就是：

$$
i-shift
$$

因为是循环数组，所以需要对 $n$ 取模。用 $1$ 下标写的时候，可以这样处理：

```c++
int real_pos(int i) {
    int pos = (i - shift) % n;
    if (pos <= 0) pos += n;
    return pos;
}
```

这样第一类操作就可以在 $O(1)$ 内完成：

1. 找到当前第 $i$ 个位置在原数组中的真实下标 $pos$；
2. 总和减去旧值；
3. 修改成新值；
4. 总和加上新值。

第二类操作只需要更新偏移量：

```c++
shift = (shift + k) % n;
```

下面是正解：

```c++
#include <bits/stdc++.h>
using namespace std;

using ll = long long;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    vector<ll> a(n + 1);
    ll sum = 0;
    for (int i = 1; i <= n; i++) {
        cin >> a[i];
        sum += a[i];
    }

    ll shift = 0;

    auto get_real = [&](int x) {
        ll pos = (x - shift) % n;
        if (pos <= 0) pos += n;
        return (int)pos;
    };

    int q;
    cin >> q;
    while (q--) {
        int op;
        cin >> op;

        if (op == 1) {
            int i;
            ll x;
            cin >> i >> x;

            int pos = get_real(i);
            sum -= a[pos];
            a[pos] = x;
            sum += a[pos];
        } else {
            ll k;
            cin >> k;
            shift = (shift + k) % n;
        }

        cout << sum << '\n';
    }

    return 0;
}
```

每次操作都只需要 $O(1)$ 的时间，所以总时间复杂度为 $O(n+q)$。数组需要保存下来，空间复杂度为 $O(n)$。
