# 货币系统

## 题目大意

给定 $n$ 种货币面额。若某种面额可以由其他更小的面额通过若干次相加得到，那么这种面额就是多余的。

要求删去所有多余面额后，最少需要保留多少种面额，使得原来能表示出的金额集合不发生变化。

## 思路分析

首先考虑一个很直接的想法：对于每个面额 $a_i$，判断它能不能由其他面额表示出来。如果能表示出来，就说明它不需要保留。

但是这里有一个重要细节：我们应该按面额从小到大处理。

因为如果一个面额 $a_i$ 可以被表示出来，那么一定只需要用比它小的面额。比它大的面额不可能参与表示 $a_i$。所以先排序，然后依次判断当前面额是否能被前面已经保留的面额表示出来即可。

这就变成了一个完全背包问题。

设：

$$
dp[j]
$$

表示当前已经保留的面额能否表示出金额 $j$。

初始时：

$$
dp[0]=true
$$

对于排序后的每个面额 $a_i$：

- 如果 $dp[a_i]=true$，说明它已经能被前面的面额表示出来，是冗余的；
- 如果 $dp[a_i]=false$，说明它必须保留，然后用完全背包更新所有能表示出的金额。

更新方式为：

```c++
for (int j = a[i]; j <= max_val; j++) {
    dp[j] |= dp[j - a[i]];
}
```

为什么是完全背包？因为每种货币可以使用任意多次。

下面是正解：

```c++
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;
    while (T--) {
        int n;
        cin >> n;

        vector<int> a(n);
        int max_val = 0;
        for (int i = 0; i < n; i++) {
            cin >> a[i];
            max_val = max(max_val, a[i]);
        }

        sort(a.begin(), a.end());

        vector<int> dp(max_val + 1, 0);
        dp[0] = 1;

        int ans = 0;

        for (int x : a) {
            if (dp[x]) {
                continue;
            }

            ans++;

            for (int j = x; j <= max_val; j++) {
                if (dp[j - x]) {
                    dp[j] = 1;
                }
            }
        }

        cout << ans << '\n';
    }

    return 0;
}
```

设最大面额为 $V$，每个面额做一次完全背包更新，时间复杂度为 $O(nV)$，空间复杂度为 $O(V)$。
