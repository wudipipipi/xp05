# 糖果游戏

## 题目大意

有 $N$ 种糖果，第 $i$ 种糖果售价为 $A_i$，购买之后会返还 $B_i$ 枚金币。糖果可以重复购买，只要当前金币数不少于售价即可。

初始有 $R$ 枚金币，问最多可以买多少颗糖果。

## 思路分析

买第 $i$ 种糖果时，虽然需要先拿出 $A_i$ 枚金币，但买完之后会返还 $B_i$ 枚金币，所以真正减少的金币数是：

$$
A_i-B_i
$$

记：

$$
d_i=A_i-B_i
$$

那么每种糖果可以看成两个属性：

- $A_i$：购买门槛，当前金币数至少要达到 $A_i$ 才能买；
- $d_i$：买完之后实际减少的金币数。

如果当前能买多种糖果，为了让后面还能买更多次，我们显然应该优先买净消耗 $d_i$ 最小的糖果。

一个比较直接的做法是：每次在所有当前能买的糖果中找 $d_i$ 最小的买一颗。这个思路是对的，但是如果 $d_i=1$，答案可能接近 $10^9$，一颗一颗模拟会超时。

下面考虑如何优化。

先把所有糖果按照售价 $A_i$ 从小到大排序。对于两种糖果 $x,y$，如果满足：

$$
A_x\le A_y
$$

并且：

$$
d_x\le d_y
$$

那么糖果 $y$ 就完全没有必要考虑。因为只要能买 $y$，一定也能买 $x$，而且 $x$ 的净消耗还不更大。

所以排序之后，我们只保留那些使得 $d_i$ 变得更小的糖果。保留下来的糖果满足：

- $A_i$ 递增；
- $d_i$ 递减。

也就是说，越贵的糖果，净消耗越小，才有保留价值。

接下来从后往前处理这些糖果。假设当前处理到一种糖果，售价为 $a$，净消耗为 $d$。如果当前金币数 $R\ge a$，那么在金币数降到 $a$ 以下之前，这种糖果就是当前能买到的最优糖果，可以连续买很多次。

最多可以买的次数为：

$$
\left\lfloor\frac{R-a}{d}\right\rfloor+1
$$

这里加 $1$ 是因为只要当前金币数仍然不少于 $a$，就还能再买一次。买完这些次之后，金币数会降到 $a$ 以下，此时这种糖果就买不了了，再继续考虑更便宜的糖果。

下面是正解：

```c++
#include <bits/stdc++.h>
using namespace std;

using ll = long long;

struct Candy {
    ll a, d;

    bool operator<(const Candy &other) const {
        if (a != other.a) return a < other.a;
        return d < other.d;
    }
};

void solve() {
    int n;
    ll r;
    cin >> n >> r;

    vector<ll> a(n), b(n);
    for (int i = 0; i < n; i++) cin >> a[i];
    for (int i = 0; i < n; i++) cin >> b[i];

    vector<Candy> all(n);
    for (int i = 0; i < n; i++) {
        all[i] = {a[i], a[i] - b[i]};
    }

    sort(all.begin(), all.end());

    // 去掉被支配的糖果，只保留前缀中 d 不断变小的位置
    vector<Candy> candy;
    ll best_d = (ll)4e18;
    for (auto x : all) {
        if (x.d < best_d) {
            best_d = x.d;
            candy.push_back(x);
        }
    }

    ll ans = 0;

    // 从净消耗最小的糖果开始处理
    for (int i = (int)candy.size() - 1; i >= 0; i--) {
        ll a = candy[i].a;
        ll d = candy[i].d;

        if (r >= a) {
            ll cnt = (r - a) / d + 1;
            ans += cnt;
            r -= cnt * d;
        }
    }

    cout << ans << '\n';
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;
    while (T--) {
        solve();
    }

    return 0;
}
```

排序复杂度为 $O(N\log N)$，过滤和贪心都是 $O(N)$，所以单组数据总时间复杂度为 $O(N\log N)$。空间复杂度为 $O(N)$。
