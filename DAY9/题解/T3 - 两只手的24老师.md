# Pair Matching 2

## 题目大意

有 $2N$ 张牌从左到右排列，每个数字 $1\sim N$ 恰好出现两次。第 $x$ 对牌配对成功可以获得 $V_x$ 分。

依次考虑每张牌，可以选择拿或不拿。手上最多会保留两张牌，可以理解为左手和右手：

- 如果新拿的牌和手中某张牌数字相同，这两张牌消失，并获得对应分数；
- 然后左手的牌会被丢弃；
- 右手的牌会转移到左手；
- 如果新拿的牌没有消失，则放入右手。

问最多能获得多少分。

## 思路分析

首先考虑朴素 DP。因为手上最多有两张牌，可以设：

$$
dp[i][l][r]
$$

表示处理到第 $i$ 张牌，左手是 $l$，右手是 $r$ 时的最大分数。每张牌考虑拿或不拿即可。

这个做法状态数是 $O(N^3)$，可以通过 $N\le 300$ 左右的部分数据，但无法处理 $N\le 4\times 10^5$。

下面考虑正解。

对于一个数字 $x$，设它的两次出现位置为：

$$
p_x<q_x
$$

如果我们要获得 $V_x$ 的分数，那么一定是在 $p_x$ 处拿起第一张牌，并在 $q_x$ 处拿起第二张牌完成配对。于是每个得分可以看成选择了一个区间：

$$
[p_x,q_x]
$$

关键是：哪些区间可以同时被选择？

因为手里最多只能保留两张未配对的牌，所以如果两个区间是嵌套关系：

$$
l_1<l_2<r_2<r_1
$$

那么外层的第一张牌会在内层配对完成时被丢弃，外层无法再配对成功。

但是交叉关系是可以的：

$$
l_1<l_2<r_1<r_2
$$

例如先拿 $l_1$，再拿 $l_2$，到 $r_1$ 时配对第一组，之后 $l_2$ 还会留在手中，最后可以在 $r_2$ 配对第二组。

所以当我们考虑当前区间 $[p,i]$ 时，若选择它，前面的结构只有两种情况：

1. 不与前一个得分区间交叉，那么之前的所有操作都在 $p$ 之前完成，贡献为：

$$
dp[p]+V_{A_i}
$$

2. 与某个区间 $[q,r]$ 交叉，其中：

$$
q<p<r<i
$$

在 $r$ 处完成 $[q,r]$ 后，位置 $p$ 的牌仍然留在手中，于是可以在 $i$ 处继续完成当前配对。此时贡献为：

$$
dp[q]+V_{A_r}+V_{A_i}
$$

我们从左到右扫描所有位置。设 $pre[i]$ 表示 $A_i$ 上一次出现的位置，如果 $pre[i]\neq 0$，那么当前位置 $i$ 是某个数字的第二次出现，可以尝试完成区间 $[pre[i],i]$。

定义：

$$
dp[i]
$$

表示只考虑前 $i$ 张牌时，能够获得的最大分数。

显然可以不选择第 $i$ 张牌：

$$
dp[i]=dp[i-1]
$$

如果 $i$ 是第二次出现，记 $p=pre[i]$，那么可以直接接在 $p$ 前面的最优方案之后：

$$
dp[i]=\max(dp[i],dp[p]+V_{A_i})
$$

还需要处理交叉情况。对于已经完成的区间 $[q,r]$，它能给后面某个 $p$ 满足 $q<p$ 的区间提供：

$$
dp[q]+V_{A_r}
$$

所以我们用树状数组维护这个值，按照区间左端点 $q$ 存进去，支持查询所有 $q<p$ 的最大值。

当处理到第二次出现位置 $i$，左端点为 $p$ 时：

$$
dp[i]=\max(dp[i],query(p-1)+V_{A_i})
$$

处理完当前区间后，再把：

$$
dp[p]+V_{A_i}
$$

加入到树状数组的下标 $p$，供后面的交叉区间使用。

下面是正解：

```c++
#include <bits/stdc++.h>
using namespace std;

using ll = long long;

struct BIT {
    int n;
    vector<ll> tree;

    BIT(int n = 0) {
        init(n);
    }

    void init(int n_) {
        n = n_;
        tree.assign(n + 1, 0);
    }

    int lowbit(int x) {
        return x & -x;
    }

    void add(int x, ll val) {
        for (; x <= n; x += lowbit(x)) {
            tree[x] = max(tree[x], val);
        }
    }

    ll query(int x) {
        ll res = 0;
        for (; x > 0; x -= lowbit(x)) {
            res = max(res, tree[x]);
        }
        return res;
    }
};

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n;
    cin >> n;

    vector<int> a(2 * n + 1);
    for (int i = 1; i <= 2 * n; i++) {
        cin >> a[i];
    }

    vector<ll> v(n + 1);
    for (int i = 1; i <= n; i++) {
        cin >> v[i];
    }

    vector<int> pre(2 * n + 1), last(n + 1, 0);
    for (int i = 1; i <= 2 * n; i++) {
        pre[i] = last[a[i]];
        last[a[i]] = i;
    }

    vector<ll> dp(2 * n + 1, 0);
    BIT bit(2 * n);

    for (int i = 1; i <= 2 * n; i++) {
        dp[i] = dp[i - 1];

        if (pre[i] != 0) {
            int p = pre[i];
            ll score = v[a[i]];

            // 当前配对不与前一个配对交叉
            dp[i] = max(dp[i], dp[p] + score);

            // 当前配对与某个已经完成的配对交叉
            dp[i] = max(dp[i], bit.query(p - 1) + score);

            // 当前配对可以作为后续区间的交叉对象
            bit.add(p, dp[p] + score);
        }
    }

    cout << dp[2 * n] << '\n';

    return 0;
}
```

每个位置只会被处理一次，树状数组每次操作是 $O(\log N)$，所以总时间复杂度为 $O(N\log N)$，空间复杂度为 $O(N)$。
