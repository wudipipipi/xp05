# Air Conditioner

## 题目大意

有一家餐厅，初始温度为 $m$。空调每分钟可以让温度升高 $1$、降低 $1$，也可以保持不变，并且可以在任意整数分钟切换状态。

现在有 $n$ 位顾客，第 $i$ 位顾客会在 $t_i$ 时刻到达，并且只接受温度在 $[l_i,r_i]$ 之间。顾客按照到达时间非递减顺序给出，问是否存在一种调节空调的方案，使得每一位顾客到达时都能满意。

## 思路分析

最直接的想法是模拟每一分钟餐厅的温度变化，但是 $t_i$ 最大可以达到 $10^9$，所以显然不能逐分钟模拟。

我们换一个角度考虑：在某一个时间点，我们并不需要知道餐厅温度具体是多少，只需要知道它可能落在哪个区间里。

一开始时间为 $0$，温度固定为 $m$，所以可能温度区间就是：

$$
[m,m]
$$

假设当前已经处理到了时间 $last$，可能温度范围是 $[x,y]$，下一位顾客在时间 $t_i$ 到达，那么中间经过了：

$$
dt=t_i-last
$$

分钟。因为每分钟温度最多变化 $1$，所以经过这段时间之后：

- 最低温度可以降到 $x-dt$；
- 最高温度可以升到 $y+dt$。

于是可能温度区间会扩展成：

$$
[x-dt,y+dt]
$$

但是第 $i$ 位顾客要求温度必须在 $[l_i,r_i]$ 中，所以此时真正可行的温度范围应该是两个区间的交集：

$$
[\max(x-dt,l_i),\min(y+dt,r_i)]
$$

如果交集为空，也就是左端点大于右端点，说明无论怎么调节空调，都无法满足当前这个顾客，答案就是 `NO`。

否则继续处理下一位顾客即可。

这个题比较容易错的地方是同一时间可能有多个顾客到达，此时 $dt=0$，温度区间不会扩展，只能继续和这些顾客的要求区间取交集。

下面是正解：

```c++
#include <bits/stdc++.h>
using namespace std;

void solve() {
    int n;
    long long m;
    cin >> n >> m;

    long long last = 0;
    long long x = m, y = m;
    bool ok = true;

    for (int i = 1; i <= n; i++) {
        long long t, l, r;
        cin >> t >> l >> r;

        long long dt = t - last;

        // 经过 dt 分钟后，当前可能温度区间向两边扩展
        x -= dt;
        y += dt;

        // 当前顾客到达时，温度还必须落在 [l,r] 中
        x = max(x, l);
        y = min(y, r);

        if (x > y) {
            ok = false;
        }

        last = t;
    }

    cout << (ok ? "YES" : "NO") << '\n';
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;
    while (T--) {
        solve();
    }

    return 0;
}
```

每一位顾客只会被处理一次，所以单组数据时间复杂度为 $O(n)$，只维护当前可行温度区间，空间复杂度为 $O(1)$。
