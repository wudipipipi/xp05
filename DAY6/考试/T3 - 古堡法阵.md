# T118343.古堡法阵

## 题目描述

输入文件：`pub.in`  
输出文件：`pub.out`

在创世纪时期的古堡中，werewolf 开设了 `N` 座魔法酒馆。然而，穿梭于酒馆之间令他疲惫不堪，城镇的古老石板路让他的行进效率低下。为此，他决定在酒馆之间架设 witch 发明的传送法阵。任何两座酒馆若被法阵相连，werewolf 便可在 `1` 秒内瞬间移动。同时有 `M` 组特定酒馆可以免费架设法阵，剩下的构筑法阵需要消耗魔法水晶：每建立一座法阵需支付 `Q` 枚水晶。fool 设计了一套评估标准：

设某方案包含 `S` 座需要付费的法阵，酒馆编号为 `1` 至 `N`。  
定义 `T(x, y)` 为从酒馆 `x` 到 `y` 通过法阵的最短时间（若无法到达则视为无穷）。  
计算所有有序酒馆对的 `T(x, y)` 之和，记为时间惩罚 `P = ∑(x=1..n) ∑(y=1..n) T(x, y)`。  
方案总成本为 `P + Q×S`。

villager 们请求你施展魔法解决此难题。

## 输入格式

第一行输入整数 `T` 表示测试用例数量（`1≤T≤1000`）。

每个测试用例包含：

1. 整数 `N`、整数 `Q`、整数 `M`，分别表示酒馆数量、法阵造价、特殊酒馆对数。
2. 接下来 `M` 行，每行两个整数 `x` 和 `y`，表示可免费连接的两个酒馆编号。

## 输出格式

对于每个测试用例，输出方案的最小成本对 `998244353` 取模。

## 说明 / 提示

### 样例解释

在测试样例 1 中，连接酒馆 `1` 和 `2`，连接酒馆 `2` 和 `3`。

时间惩罚计算方式为：

`T(1,1)=0, T(1,2)=1, T(1,3)=2`

`T(2,1)=1, T(2,2)=0, T(2,3)=1`

`T(3,1)=2, T(3,2)=1, T(3,3)=0`

因此时间惩罚 `P=8`，法阵总花费为 `2×2=4`，总花费为 `12`。
