# 跳题

## 题目描述

已经是3024年了，问题的创意早已枯竭，现在算法竞赛采用了一种改良的个人赛形式。竞赛包含$n$道题目，编号从$1$到$n$。第$i$道题目有其对应的分数$a_i$和特定参数$b_i$（$1 \le b_i \le n$）。

最初，测试系统会提供给参赛者**第一道**题目。当参赛者拿到第$i$道题目时，他们有两种选择：

- **提交**该题目并获得$a_i$分；
- **跳过**该题目，之后将永远无法提交它。

然后，测试系统会从满足以下条件的题目中为参赛者选择下一道题目：

- 如果他**提交**了第$i$道题目，系统会查看编号$j < i$的题目；
- 如果他**跳过**了第$i$道题目，系统会查看编号$j \leq b_i$的题目。

在这些题目中，系统会选择**未被提供过**（既未提交也未跳过）的**最大编号**的题目。如果没有这样的题目，参赛者的比赛将**结束**，其得分等于所有提交题目的分数之和。特别地，如果参赛者提交了第一道题目，比赛将立即结束。注意，每道题目最多只会被提供给参赛者**一次**。

你为这次竞赛做了充分准备，现在你可以提交任何题目。请确定你在竞赛中能够获得的最大分数。

## 输入格式

每个测试包含多个测试用例。第一行包含一个整数 $t$（$1 \leq t \leq 10^5$）——测试用例的数量。接下来是每个测试用例的描述。

每个测试用例的第一行包含一个整数 $n$（$1 \leq n \leq 4 \cdot 10^5$）——竞赛中题目的数量。

每个测试用例的第二行包含 $n$ 个整数 $a_1, a_2, \ldots, a_n$（$1 \leq a_i \leq 10^9$）——每道题目的分数。

每个测试用例的第三行包含 $n$ 个整数 $b_1, b_2, \ldots, b_n$（$1 \leq b_i \leq n$）——每道题目的参数。

保证所有测试用例的 $n$ 之和不超过 $4 \cdot 10^5$。

## 输出格式

对于每个测试用例，输出一个整数 —— 能够获得的最大分数。


### 输入样例1

```
4
2
15 16
2 1
5
10 10 100 100 1000
3 4 1 1 1
3
100 49 50
3 2 2
4
100 200 300 1000
2 3 4 1
```

### 输出样例1

```
16
200
100
1000
```

## 说明/提示


可以选择跳过第一题；然后他会收到编号为 $b_1 = 2$ 的题目。他可以提交该题并获得 $a_2 = 16$ 分。之后比赛结束，因为 Prokhor 已经收到了所有题目。注意，如果提交第一题，他会获得 $a_1 = 15$ 分，但比赛会立即结束。


可以跳过第一题；然后会收到编号为 $b_1 = 3$ 的题目。可以提交该题并获得 $a_3 = 100$ 分。之后，会收到第二题，可以选择跳过它，从而收到编号为 $b_2 = 4$ 的题目。可以提交第四题并再获得 $a_4 = 100$ 分。之后比赛结束，因为已经收到了所有编号不超过 $4$ 的题目。因此，总共会获得 $200$ 分。


可以直接提交第一题并获得 $100$ 分，之后比赛立即结束。




对于所有测试点都满足 $1 \leq t \leq 10^5$，$1 \leq n \leq 4 \cdot 10^5$，$1 \leq a_i \leq 10^9$，$1 \leq b_i \leq 10^9$。

| $1\sim 2$  |     $2\leq n \leq 1000$      |          |

| $4\sim 10$ | $2\leq n \leq 4 \cdot 10^5$  |          |

## 思路分析

**注意本题数据除了测试点5和6全部卡掉了INT_MAX**

首先我们思考一下对于一个题来说，到达这个题之后向前一直做题能拿到多少，不难想到我们跳到这里代表舍弃了一些题，而我们包括这个题之前所有的题的分数，减去跳到这里的最小花费，就可以求出当前题最高能拿多少分了。

对于一个具体的题来说，在它之前有多少分数是好算的，这个就是个前缀和。而对于跳到某个位置具体要舍弃掉多少分，我们考虑一个 $\text{DP}$，我们设计 $dp[i]$ 的意义为跳到第 $i$ 题之后一直向前做的最大得分。我们枚举每一个 $i$，枚举 $j$ 从 $i+1$，到 $b[i]$，因为我们求的是最小值，所以状态转移方程显然是 $dp[j] = \min(dp[j],dp[i]+dp[j])$，这个解法的时间复杂度是 $O(n^2)$，可以通过前四个测试点。

```c++
void solve() {
    int n;
    cin >> n;
    vector<long long> a(n + 1), b(n + 1), pre(n + 1, 0);
    vector<long long> dp(n + 1, LLONG_MAX);
    vector<long long> vis(n + 1, 0);
    long long sum = 0;
    for (long long i = 1; i <= n; ++i) {
        cin >> a[i];
        sum += a[i];
        pre[i] = sum;
    }
    for (int i = 1; i <= n; ++i) {
        cin >> b[i];
    }
    long long res = a[1];
    dp[1] = 0;
    for (int i = 1; i <= n; ++i) {
        for (int j = i + 1; j <= b[i]; ++j) {
            dp[j] = min(dp[j], dp[i] + a[i]);
        }
    }
    for (int i = 1; i <= n; ++i) {
        res = max(res, pre[i] - dp[i]);
    }
    cout << res << endl;
}
```

对于如何优化，我们发现上述的 $\text{DP}$ 中许多转移都是无意义的，而从某个题跳到某个题这种方式可以想到连边建图，那么对于我们要求的东西实际上就是抵达某个点的最短路。因此使用 $\text{dijkstra}$ 的堆优化版本，可以达到 $O(m\log n)$ 的复杂度，其中 $m$ 和 $n$ 等阶。

```c++
typedef pair<long long, long long> PLL;
void solve() {
    int n; cin >> n;
    vector<long long> vec(n);
    for(auto &x : vec) cin >> x;
    vector<long long> b(n);
    for(auto &x : b) cin >> x,x--;
    vector<long long> dis(n, LLONG_MAX);
    priority_queue<PLL, vector<PLL>, greater<PLL>> pq;
    pq.push({0, 0});
    while (!pq.empty()) {
        long long d = pq.top().first;
        long long i = pq.top().second;
        pq.pop();
        if (dis[i] != LLONG_MAX) continue;
        dis[i] = d;
        pq.push({d + vec[i], b[i]});
        if (i > 0) pq.push({d, i - 1});
    }
    long long ans = 0;
    long long sum = 0;
    for (int i = 0; i < n; i++) {
        sum += vec[i];
        ans = max(ans, sum - dis[i]);
    }
    cout << ans << endl;
}
```
