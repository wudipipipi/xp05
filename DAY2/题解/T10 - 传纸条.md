# 传纸条

## 题目大意

给定一个 $m\times n$ 的矩阵，每个格子有一个好感度。

需要从左上角到右下角走一条路径，只能向右或向下；同时还需要从右下角回到左上角走一条路径，只能向左或向上。除了起点和终点外，同一个同学不能在两次传纸条中都帮忙。

要求两条路径经过格子的好感度之和最大。

## 思路分析

直接考虑“一条从左上到右下，一条从右下到左上”的来回路径不太方便。我们可以把第二条路径反过来看：从右下走回左上，只能向上或向左；反过来就是从左上走到右下，只能向下或向右。

于是问题变成：

> 两个人同时从左上角走到右下角，每一步都只能向下或向右，并且除了起点和终点外不能走到同一个格子，求经过格子权值和最大。

这样就可以做 DP。

设：

$$
dp[x_1][y_1][x_2][y_2]
$$

表示第一个人走到 $(x_1,y_1)$，第二个人走到 $(x_2,y_2)$ 时，能够获得的最大好感度。

因为两个人是同时走的，所以他们走过的步数必须相同。也就是：

$$
x_1+y_1=x_2+y_2
$$

不满足这个条件的状态不合法。

对于一个合法状态，它可以由四种前驱转移而来：

1. 两个人都从上方走下来；
2. 第一个人从上方走下来，第二个人从左边走过来；
3. 第一个人从左边走过来，第二个人从上方走下来；
4. 两个人都从左边走过来。

也就是取这四种前驱的最大值。

当前格子的贡献为：

- 先加上 $(x_1,y_1)$ 的好感度；
- 如果 $(x_1,y_1)$ 和 $(x_2,y_2)$ 不是同一个格子，再加上 $(x_2,y_2)$ 的好感度。

这样可以避免同一个格子被重复计算。

下面是正解：

```c++
#include <bits/stdc++.h>
using namespace std;

int a[55][55];
int dp[55][55][55][55];

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int m, n;
    cin >> m >> n;

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            cin >> a[i][j];
        }
    }

    dp[1][1][1][1] = a[1][1];

    for (int x1 = 1; x1 <= m; x1++) {
        for (int y1 = 1; y1 <= n; y1++) {
            for (int x2 = 1; x2 <= m; x2++) {
                for (int y2 = 1; y2 <= n; y2++) {
                    if (x1 == 1 && y1 == 1 && x2 == 1 && y2 == 1) {
                        continue;
                    }

                    // 两个人必须走了相同步数
                    if (x1 + y1 != x2 + y2) {
                        continue;
                    }

                    int best = 0;

                    if (x1 > 1 && x2 > 1) {
                        best = max(best, dp[x1 - 1][y1][x2 - 1][y2]);
                    }
                    if (x1 > 1 && y2 > 1) {
                        best = max(best, dp[x1 - 1][y1][x2][y2 - 1]);
                    }
                    if (y1 > 1 && x2 > 1) {
                        best = max(best, dp[x1][y1 - 1][x2 - 1][y2]);
                    }
                    if (y1 > 1 && y2 > 1) {
                        best = max(best, dp[x1][y1 - 1][x2][y2 - 1]);
                    }

                    int val = a[x1][y1];
                    if (x1 != x2 || y1 != y2) {
                        val += a[x2][y2];
                    }

                    dp[x1][y1][x2][y2] = best + val;
                }
            }
        }
    }

    cout << dp[m][n][m][n] << '\n';

    return 0;
}
```

状态数为 $O(m^2n^2)$，每个状态有常数个转移，所以时间复杂度为 $O(m^2n^2)$，空间复杂度也是 $O(m^2n^2)$。在 $m,n\le 50$ 的限制下可以通过。
