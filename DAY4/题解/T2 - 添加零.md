# 添加零

## 题目描述

给定一个初始包含 $n$ 个整数的数组 $a$。每次操作中，你需要执行以下步骤：

- 选择一个位置 $i$，满足 $1 < i \le |a|$ 且 $a_i = |a| + 1 - i$，其中 $|a|$ 表示数组当前的**长度**。
- 在数组末尾追加 $i - 1$ 个零。

在可以执行任意次操作后，数组 $a$ 的最大可能长度是多少？

## 输入格式

每个测试包含多个测试用例。第一行包含测试用例的数量 $t$（$1 \le t \le 1000$）。接下来是每个测试用例的描述。

每个测试用例的第一行包含一个整数 $n$（$1 \le n \le 3 \cdot 10^5$）—— 数组 $a$ 的长度。

每个测试用例的第二行包含 $n$ 个整数 $a_1, a_2, \ldots, a_n$（$1 \le a_i \le 10^{12}$）。

保证所有测试用例的 $n$ 之和不超过 $3 \cdot 10^5$。

## 输出格式

对于每个测试用例，输出一个整数 —— 在执行任意次操作后，数组 $a$ 可能达到的最大长度。


### 输入样例1

```
4
5
2 4 6 2 5
5
5 4 4 5 1
4
6 8 2 3
1
1
```

### 输出样例1

```
10
11
10
1
```

## 说明/提示

#### 样例解释

在第一个测试用例中，我们可以先选择 $i = 4$，因为 $a_4 = 5 + 1 - 4 = 2$。操作后数组变为 $[2, 4, 6, 2, 5, 0, 0, 0]$。接着可以选择 $i = 3$，因为 $a_3 = 8 + 1 - 3 = 6$。操作后数组变为 $[2, 4, 6, 2, 5, 0, 0, 0, 0, 0]$，长度为 $10$。可以证明没有任何操作序列能使最终数组更长。

在第二个测试用例中，我们可以依次选择 $i=2$、$i=3$ 和 $i=4$。最终数组将变为 $[5, 4, 4, 5, 1, 0, 0, 0, 0, 0, 0]$，长度为 $11$。

#### 数据范围

对于所有测试数据，都满足 $1 \le t \le 10^4$，$1 \le n \le 3 \cdot 10^5$，$1 \le a_i \le 10^{12}$，$\displaystyle\sum n \leq 3 \cdot 10^5$

|   测试点    |          $n$          |       $a_i$        |
| :---------: | :-------------------: | :----------------: |
|  $1\sim 2$  |      $n \leq 5$       | $a_i \leq 10^{12}$ |
|  $3\sim 6$  |  $\sum n \leq 3000$   | $a_i \leq 10^{12}$ |
| $7\sim 10$  | $n \leq 3 \cdot 10^5$ |  $a_i \leq 10^6$   |
| $11\sim 20$ | $n \leq 3 \cdot 10^5$ | $a_i \leq 10^{12}$ |

## 题目大意

给定一个初始包含 $n$ 个整数的数组 $a$。每次操作中，你需要执行以下步骤：

- 选择一个位置 $i$，满足 $1 < i \le |a|$ 且 $a_i = |a| + 1 - i$，其中 $|a|$ 表示数组当前的**长度**。
- 在数组末尾追加 $i - 1$ 个零。

在可以执行任意次操作后，数组 $a$ 的最大可能长度是多少？

## 思路分析

因为一个数据被使用之后，数组长度变大，不可能再使用这个数据，所以对于 $n \leq 5$ 可以使用全排列。

```c++
typedef long long ll;
int sum_n;
void solve() {
    int n;
    cin >> n;
    sum_n += n;
    std::vector<ll> a(n + 1);
    for (int i = 1; i <= n; i++) {
        cin >> a[i];
    }
    vector<int> v;
    for (int i = 1; i <= n; i++) {
        v.push_back(i);
    }
    ll maxx = n;
    do {
        ll len = n;
        for (int i = 0; i < n; i++) {
            int now = v[i];
            if (a[now] == len + 1 - now) {
                len += now - 1;
            }
        }
        maxx = max(maxx, len);
    } while (next_permutation(v.begin(), v.end()));
    cout << maxx << endl;
}
```

还存在一个 $O(n^2)$ 的 $\text{DFS}$ 写法，但是这个写法在测试点 $7$ 到 $10$ 上依然可以跑的特别快，因为限制了 $a_i$ 的值域，我们确保搜索的轮数不会特别大，所以这个做法其实可以拿到 $50$ 分：

```c++
const int N = 5e5;
vector<long long>v(N);
long long ans = 0;
map<long long,int>mp;
int n;
void dfs(int x) {
    for (int i = 2 ; i <= n ; i ++) {
      if (v[i] == x - i + 1) {
          dfs(x + i - 1);
      }  
    }
    ans = max(ans , 1LL*x);
}
void solve() {
    cin >> n;
    ans = 0;
    for (int i = 1 ; i <= n ; i ++) {
        cin >> v[i];
    }
    dfs(n);
    cout << ans << endl;
}
```

下面是正解：

看到这个式子，变形，把 $a_i=|a|+1-i$ 移项为 $a_i+i-1=|a|$，左边这个式子可以直接在输入的时候处理出来，我们记录其为 $b_i$，那么实际上在这个题里我们只需要关心 $b_i$ 和其对应的 $i$ 就可以了，我们用 $\text{pair<int,int>}$ 把这两个东西绑定在一起。

我们对于这些 $\text{pair}$ 排序，维护一个 $\text{set}$ 来记录可能达到的数组长度，一开始插入一个 $n$。

思考一下这个东西的本质，其实对于一个较大的 $b_i$ 来说，我们有可能需要通过先加上一个较小的长度才能够让 $b_i=|a|$，因此我们排序之后，从小到大枚举，看 $\text{set}$ 中是否有对应的 $|a|$，因为我们只关心长度，不关心是如何得出来的，所以用 $\text{set}$ 只要找到一个对应的数字，就可以把它加 $i-1$ 插入到 $\text{set}$ 中，这样一来最后输出 $\text{set}$ 中最大的数字就可以了。

```c++
#define int long long
void solve() {
    int n;
    cin >> n;
    vector<pair<int,int>> vec(n + 1);
    set<int> st;
    for (int i = 1; i <= n; ++i) {
        cin >> vec[i].first;
        vec[i].first += i - 1;
        vec[i].second = i;
    }
    sort(vec.begin() + 1, vec.end());
    st.insert(n);
    for (int i = 1; i <= n; ++i) {
        if (st.count(vec[i].first)) {
            st.insert(vec[i].first + vec[i].second - 1);
        }
    }
    cout << *st.rbegin() << endl;
}
```
