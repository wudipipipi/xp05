# 跳跃机器人

## 题目大意

有 $n$ 个平台围成一个环，第 $i$ 个平台跳到下一个平台需要机器人的灵敏度至少为 $d_i$。机器人每跳一次，灵敏度会增加 $1$。

现在需要选择一个起始平台和一个初始灵敏度 $a$，让机器人恰好跳 $n$ 次后回到起点。要求输出最小的初始灵敏度，以及对应的起始平台编号。

当 $f=1$ 时，题目直接给出 $d_i$；当 $f=2$ 时，后面的 $d_i$ 需要按照递推式生成。

## 思路分析

首先考虑暴力做法。我们可以枚举起点，然后模拟从这个起点开始跳一圈的过程，计算这个起点需要的最小初始灵敏度。这样每个起点需要 $O(n)$，一共有 $n$ 个起点，总复杂度是 $O(n^2)$。

这个做法可以通过 $n\le 300$ 的子任务 $0$，以及 $n\le 5000$ 的子任务 $1$，一共可以拿到 $15+17=32$ 分。如果题目保证从第一个平台开始跳是最佳选择，那么只计算起点 $1$ 的答案即可，可以拿到子任务 $2$ 和子任务 $4$ 的 $10+5=15$ 分。

下面考虑正解。

假设我们选择平台 $s$ 作为起点。机器人第一次跳的时候灵敏度是 $a$，第二次跳的时候是 $a+1$，第 $k+1$ 次跳的时候就是 $a+k$。

如果把环拆开复制一遍，那么从平台 $s$ 开始跳一圈，需要依次经过：

$$
d_s,d_{s+1},\dots,d_{s+n-1}
$$

对于其中第 $k+1$ 条边，需要满足：

$$
a+k\ge d_{s+k}
$$

也就是：

$$
a\ge d_{s+k}-k
$$

所以如果起点是 $s$，它需要的最小初始灵敏度就是：

$$
\max_{k=0}^{n-1}(d_{s+k}-k)
$$

我们把下标换成 $i=s+k$，那么 $k=i-s$，上式变成：

$$
\max_{i=s}^{s+n-1}(d_i-(i-s))
$$

也就是：

$$
s+\max_{i=s}^{s+n-1}(d_i-i)
$$

于是问题转化成：对于每个长度为 $n$ 的窗口，求窗口内 $d_i-i$ 的最大值，然后取所有起点中的最小答案。

这是一个经典的滑动窗口最大值问题，可以用单调队列 $O(n)$ 解决。

我们维护一个队列，队列中下标对应的 $d_i-i$ 从大到小排列。每次右端点向右移动时：

1. 把已经不在当前长度为 $n$ 的窗口内的下标弹出；
2. 把队尾所有不大于当前 $d_i-i$ 的下标弹出；
3. 把当前下标加入队列；
4. 当窗口长度达到 $n$ 时，队首就是当前窗口的最大值。

需要注意的是，本题 $n$ 最大到 $10^7$，所以数组要开得比较大。另外在 $f=2$ 生成数组时，$x\cdot d_{i-2}+y\cdot d_{i-1}+z$ 可能超过 `int`，这里要使用 `long long` 计算。

下面是正解：

```c++
#include <bits/stdc++.h>
using namespace std;

using ll = long long;

const int N = 10000000 + 5;
int d[2 * N], c[100000 + 5];

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, f;
    cin >> n >> f;

    if (f == 1) {
        for (int i = 1; i <= n; i++) {
            cin >> d[i];
        }
    } else {
        int m;
        ll x, y, z;
        cin >> m >> x >> y >> z;

        for (int i = 1; i <= m; i++) {
            cin >> c[i];
            d[i] = c[i];
        }

        const ll mod = 1000000000;
        for (int i = m + 1; i <= n; i++) {
            d[i] = ((x * d[i - 2] + y * d[i - 1] + z) % mod) + 1;
        }
    }

    // 复制一遍，把环形问题变成线性区间问题
    for (int i = 1; i <= n; i++) {
        d[i + n] = d[i];
    }

    deque<int> q;
    int ans = INT_MAX;
    int ans_pos = 1;

    for (int i = 1; i <= 2 * n - 1; i++) {
        // 当前窗口左端点为 i - n + 1，长度不能超过 n
        while (!q.empty() && q.front() <= i - n) {
            q.pop_front();
        }

        while (!q.empty() && d[q.back()] - q.back() <= d[i] - i) {
            q.pop_back();
        }
        q.push_back(i);

        if (i >= n) {
            int s = i - n + 1;
            int need = s + d[q.front()] - q.front();

            if (need < ans) {
                ans = need;
                ans_pos = s;
            }
        }
    }

    cout << ans << ' ' << ans_pos << '\n';

    return 0;
}
```

生成数组需要 $O(n)$，单调队列处理所有窗口也是 $O(n)$，所以总时间复杂度为 $O(n)$。空间复杂度为 $O(n)$。
