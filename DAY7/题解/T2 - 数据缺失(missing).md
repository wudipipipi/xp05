# 数据缺失

## 题目大意

有一个长度为 $n$ 的序列 $a$。对于每一对下标 $i<j$，都会得到一个数：

$$
f_{i,j}=\max(a_i,a_j)
$$

原本一共有 $\frac{n(n-1)}2$ 个这样的数，现在少了其中一个。给出剩下的所有数，要求判断缺失的数是什么。

如果缺失的数只有一种可能，输出这个数；如果有无限多种可能，输出 `Infinite`。

## 思路分析

首先考虑完整情况下，所有 $\max(a_i,a_j)$ 会有什么规律。

把原数组 $a$ 从大到小排序，设排序后为：

$$
b_1\ge b_2\ge \cdots \ge b_n
$$

那么 $b_1$ 是最大值，它和其他 $n-1$ 个数配对时，较大值都是 $b_1$，所以 $b_1$ 会在所有 $f_{i,j}$ 中出现 $n-1$ 次。

接着看 $b_2$。它和 $b_1$ 配对时，最大值是 $b_1$，不会贡献 $b_2$；它和后面的 $n-2$ 个数配对时，最大值都是 $b_2$。所以 $b_2$ 会出现 $n-2$ 次。

同理，完整数据从大到小分组后，应该是：

- 最大的一组出现 $n-1$ 次；
- 第二大的一组出现 $n-2$ 次；
- 第三大的一组出现 $n-3$ 次；
- ...
- 最后一组出现 $1$ 次。

现在只缺失了一个数，所以我们可以把给出的数据排序，然后从大到小按照这些频次去匹配。

例如当前应该匹配 $cnt$ 个相同的数，如果这 $cnt$ 个位置不够，或者这 $cnt$ 个数并不全相同，就说明缺失的数就在当前这一组里，答案就是当前看到的这个值。

如果所有能检查到的组都没有问题，那么缺失的数据可能是原数组中最小值相关的那一部分。由于 $a$ 中可以出现不在输入范围内的数，并且最小值本身不会影响任何 $\max$ 的结果，所以此时有无限多种可能，输出 `Infinite`。

下面是正解：

```c++
#include <bits/stdc++.h>
using namespace std;

void solve() {
    long long n;
    cin >> n;

    long long m = n * (n - 1) / 2 - 1;
    vector<long long> v(m);
    for (int i = 0; i < m; i++) {
        cin >> v[i];
    }

    sort(v.begin(), v.end());

    long long need = n - 1;
    long long r = m - 1;

    while (r >= 0) {
        long long l = r - need + 1;

        // 当前这一组本来应该有 need 个相同的数
        if (l < 0 || v[l] != v[r]) {
            cout << v[r] << '\n';
            return;
        }

        r = l - 1;
        need--;
    }

    cout << "Infinite\n";
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int T;
    cin >> T;
    while (T--) {
        solve();
    }

    return 0;
}
```

排序复杂度为 $O(n^2\log n)$，因为输入规模本身就是 $\frac{n(n-1)}2-1$。在本题 $\sum n\le 1000$ 的限制下可以通过。空间复杂度为 $O(n^2)$。
