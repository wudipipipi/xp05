# Robin Hood Archery

## 题目大意

给定一个长度为 $n$ 的数组 $a$，每次询问一个区间 $[l,r]$。只保留这个区间里的目标，Robin Hood 和 Sheriff 轮流选择一个还没被选择过的目标，Robin Hood 先手，选到目标后获得对应分数。

两个人都会尽可能让自己的得分更高。对于每个询问，判断 Sheriff 是否不会输，也就是 Sheriff 最后的得分是否能大于等于 Robin Hood。

## 思路分析

首先考虑一个固定的区间。如果把区间里的分数从大到小排序，那么游戏中每个人的最优策略一定是每次拿当前最大的数。

因为所有目标的分数都是正数，如果当前有一个最大值 $x$，但是当前玩家不拿它，而去拿一个更小的 $y$，那么对手下一步就可以拿走 $x$，当前玩家显然不会更优。所以整个过程等价于：

- Robin Hood 拿排序后第 $1,3,5,\dots$ 个数；
- Sheriff 拿排序后第 $2,4,6,\dots$ 个数。

由于排序后有：

$$
b_1 \ge b_2 \ge b_3 \ge b_4 \ge \cdots
$$

所以每一组 $(b_1,b_2),(b_3,b_4),\dots$ 中，Robin Hood 拿到的数都不小于 Sheriff 拿到的数。也就是说 Sheriff 最多只能打平，不可能严格赢。

那么什么时候能打平呢？

只有当每一组两个数都相等，并且区间长度不能为奇数。换句话说，区间里每一种分数都必须出现偶数次。

所以问题就转化成：

对于每个询问 $[l,r]$，判断这个区间内是否所有数字的出现次数都是偶数。

如果暴力统计每个询问中的出现次数，复杂度会达到 $O(nq)$，肯定不行。我们需要快速判断区间内每个数的奇偶性。

这里可以使用随机哈希。给每一个可能出现的数 $x$ 随机分配一个权值 $val_x$，然后对数组做前缀异或：

$$
pre_i=pre_{i-1}\oplus val_{a_i}
$$

对于一个区间 $[l,r]$，它的异或值就是：

$$
pre_r \oplus pre_{l-1}
$$

异或有一个非常好的性质：相同的数异或两次会抵消掉。所以如果区间内每个数字都出现偶数次，那么这个区间异或值一定为 $0$。

反过来，如果区间内存在某些数字出现奇数次，那么理论上有可能因为随机哈希碰撞导致异或值也为 $0$，但使用 $64$ 位随机数时概率极低，可以认为不会发生。

下面是正解：

```c++
#include <bits/stdc++.h>
using namespace std;

using ull = unsigned long long;

const int N = 1000000 + 10;
ull h[N];

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    mt19937_64 rng(chrono::steady_clock::now().time_since_epoch().count());
    for (int i = 1; i < N; i++) {
        h[i] = rng();
    }

    int T;
    cin >> T;
    while (T--) {
        int n, q;
        cin >> n >> q;

        vector<ull> pre(n + 1, 0);
        for (int i = 1; i <= n; i++) {
            int x;
            cin >> x;
            pre[i] = pre[i - 1] ^ h[x];
        }

        while (q--) {
            int l, r;
            cin >> l >> r;

            ull cur = pre[r] ^ pre[l - 1];
            if (cur == 0) {
                cout << "YES\n";
            } else {
                cout << "NO\n";
            }
        }
    }

    return 0;
}
```

预处理每个测试用例需要 $O(n)$，每个询问只需要 $O(1)$ 判断，所以总时间复杂度为 $O(n+q)$。空间复杂度为 $O(n+V)$，其中 $V$ 是值域大小，本题中 $a_i\le 10^6$。
